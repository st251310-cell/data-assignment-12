進行ガイド with Gemini

テンプレートから自分のリポジトリを作り、先生を招待して、Geminiの思考モードを相棒にCSVデータを分析する。この1ページで、提出までの全手順とプロンプト例をガイドします。

STEP 1テンプレートから自分のリポジトリを作る

  1. 先生のリポジトリのページを開き、右上の緑色の Use this template → Create a new repository を押す。
  2. Repository name に data-assignment-出席番号 と入力する(例:data-assignment-07)。
  3. 公開設定(Public / Private)は先生の指示に従い、Create repository を押す。
  4. 自分のリポジトリができたら、data/score.csv を開き、Download raw file(ダウンロードアイコン)から手元に保存する。このファイルをGeminiに渡します。
テンプレートとフォークの違いは?テンプレートから作ったリポジトリは、元のリポジトリとつながりのない独立したコピーです。履歴もまっさらな状態から始まります。だからこそ、次のSTEP 2で先生を招待しないと、先生は君の提出物を見ることができません。

STEP 2先生をコラボレーターに招待する

  1. 作成した自分のリポジトリで Settings タブを開く。
  2. 左メニューの Collaborators(Collaborators and teams)を選ぶ。パスワードや2段階認証の確認を求められたら入力する。
  3. Add people を押し、先生のユーザー名 GreatT0511 を検索して Add ... to this repository で招待を送る。
  4. 「Pending invite(招待中)」と表示されれば送信完了。先生が承認すると確定します。
⚠ この手順は提出の必須条件招待を忘れると、課題をどれだけ完璧に仕上げても先生は採点できません。リポジトリを作ったら、課題に取りかかる前にまず招待を済ませること。

STEP 3Geminiの思考モードを準備する

  1. Gemini(アプリまたはブラウザ)を開き、ログインする。
  2. 画面のモデル選択から、「Thinking(思考)」対応のモデルを選ぶ。じっくり推論してから答えるモードで、統計の計算やグラフ作成のような課題に向いています。
  3. クリップ(+)アイコンから、ダウンロードした score.csv をアップロードする。
⚠ 画面は変わることがあるGeminiのメニュー名やモデル名はアップデートで変わることがあります。見つからないときは「ファイルを添付できて、考える過程を表示するモデル」を探してください。困ったら先生に質問を。

STEP 4Geminiと一緒に分析する(課題1)

CSVをアップロードしたら、次のようなプロンプトを送ってみましょう。ポイントは、答えだけでなく「求め方」も説明させることです。

PROMPT — 課題1:代表値
添付したCSVは、あるクラス40人の「学習時間」「睡眠時間」「テスト得点」のデータです。
「テスト得点」について、平均値・中央値・最大値・最小値・標準偏差を求めてください。
それぞれの値が何を意味するのか、高校生にわかる言葉で説明も加えてください。
計算にはコード実行を使い、使ったコードも見せてください。
⚠ 必ず検算するAIは計算を間違えることがあります。少なくとも平均値と最大値・最小値は、表計算ソフト(AVERAGE、MAX、MIN 関数)で自分でも計算して、一致するか確かめてからREADME.mdに記入しましょう。検算してはじめて「自分の答え」になります。

STEP 5グラフを作る(課題2・3)

グラフの作り方は2つのルートがあります。どちらでもOKです。

ルートA:Geminiに直接作らせる下のプロンプトでグラフを生成させ、表示されたグラフを画像として保存する(右クリック保存、またはスクリーンショット)。
ルートB:コードを書かせて自分で実行GeminiにPythonコードを書かせて、Google Colabに貼り付けて実行する。グラフを右クリックで保存できる。うまく動かないときはエラー文をGeminiに貼って質問しよう。
PROMPT — 課題2:ヒストグラム
このCSVの「テスト得点」のヒストグラムを作ってください。
条件:
- 階級の幅は10点(0-10, 10-20, ... 90-100)
- タイトルは「テスト得点の分布」、横軸は「得点」、縦軸は「人数」
- 日本語が文字化けする場合は japanize-matplotlib を使う
Pythonのコードも一緒に見せてください。
PROMPT — 課題3:散布図と相関係数
次の2つの散布図を別々に作ってください。
1. 横軸「学習時間」、縦軸「テスト得点」
2. 横軸「睡眠時間」、縦軸「テスト得点」
それぞれにタイトルと軸ラベルをつけてください。
また、2つの組み合わせの相関係数を小数第2位まで求めてください。
相関係数の値から「強い相関・弱い相関・ほぼ無相関」のどれにあたるかも教えてください。

STEP 6コミットして提出する

  1. 自分のリポジトリで output フォルダを開き、Add file → Upload files から3枚のpng画像をアップロードする。
  2. コミットメッセージに「ヒストグラムと散布図を追加」のように、何をしたか分かる日本語を書いてコミットする。
  3. README.md を開いて鉛筆アイコン(Edit)を押し、解答欄を書き換えてコミットする。
  4. READMEのプレビューで、表の数値と3枚のグラフ画像がきちんと表示されているか確認する。画像が表示されないときは、下の「画像を表示させるしくみ」を読んで直す。
  5. 自分のリポジトリのURLを、先生の指定した方法で提出する。
コミットは最低3回「課題1を記入」「グラフを追加」「考察を記入」のように、作業の区切りごとにコミットしましょう。コミット履歴も評価の対象です。

GUIDE画像を表示させるしくみ

READMEにグラフを表示させるには、「画像ファイルがどこにあるか(パス)」を文字で指定します。コピペで貼り付けるのではなく、場所を書いて読み込ませるのがポイントです。書き方は2通りあります。

書き方① Markdown記法(おすすめ・課題のREADMEはこちら)

MARKDOWN — README.md に書く
![ヒストグラム](output/histogram.png)
  • ! … 「画像を表示する」という合図。これがないとただのリンクになる。
  • [ヒストグラム] … 画像が表示できないときに代わりに出る説明文。
  • (output/histogram.png) … 画像ファイルの場所。カッコの中にスペースを入れないこと。

書き方② HTMLのimgタグ(大きさを変えたいとき)

HTML — README.md やHTMLファイルに書く
<img src="output/histogram.png" alt="ヒストグラム" width="480">
  • src … 画像の場所(source)。Markdownのカッコの中と同じもの。
  • alt … 代わりの説明文。Markdownの [ ] と同じ役割。
  • width … 横幅をピクセルで指定。Markdown記法ではできない、HTMLだけの機能。

「output/histogram.png」ってどういう意味?

パスは「README.mdから見た道順」です。リポジトリの中身がこうなっているとき:

リポジトリのフォルダ構成
data-assignment-07/
├── README.md          ← この中に画像の場所を書く
├── data/
│   └── score.csv
└── output/
    └── histogram.png  ← 表示させたい画像

README.mdから見ると、画像は「output フォルダの中の histogram.png」にあるので、道順は output/histogram.png となります。フォルダの区切りは /(スラッシュ)で書きます。

表示されない!ときの原因トップ4

症状・原因直し方
画像をまだコミットしていないパスを書く前に、pngをoutputフォルダにアップロードしてコミットする。GitHub上にないファイルは表示できない。
ファイル名が1文字でも違うhistgram.png(綴りミス)や Histogram.png はNG。大文字と小文字も区別される。outputフォルダを開いて実際の名前をコピーするのが確実。
拡張子が違う保存した画像が .jpg や .PNG になっていないか確認。パスの拡張子と完全に一致させる。
全角文字が混ざっているカッコやスラッシュ、スペースが全角(()/ )になっていると読み込めない。すべて半角で書く。
確認のコツREADME.mdの編集画面で Preview タブを押すと、コミットする前に表示を確認できます。画像が出ない場合は、上の表を上から順にチェック。

RULESAIを使うときの約束

この課題では、Geminiは「計算とグラフ作成を手伝ってくれる相棒」です。考えるのは君自身です。

  • 考察(課題4)を丸投げしない。AIの答えをコピペした文章は、読めばすぐ分かります。AIに「相関と因果の違いを説明して」と質問して学ぶのはOK。最後は必ず自分の言葉で書くこと。
  • 数値は必ず検算する。AIの出した値をそのまま信じない。表計算ソフトで確かめる。
  • 使ったプロンプトを記録する。READMEの最後に「使用したAIと主なプロンプト」を1〜2行で書き添えること。AIの利用は禁止ではなく、正直に書くのがルールです。
  • 個人情報を入力しない。本名・学校名・友だちの情報などをAIに送らないこと。
